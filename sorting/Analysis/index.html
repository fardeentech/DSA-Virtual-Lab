Running Time of Insertion Sort

Insertion Sort sorts an array of N elements by building a sorted portion of the array one element at a time. In each iteration, an element is taken and inserted into its correct position among the already sorted portion of the array.

The time complexity of inserting a single element into the sorted portion of the array is O(N) in the worst case (when the element needs to be compared with all previously sorted elements). Since this insertion operation is performed for every element in the array, the overall time complexity of Insertion Sort becomes O(N2).

Best and Worst Cases for Insertion Sort

  • Best Case: The array is already sorted. Every element will be compared once and inserted immediately into its correct position without any additional comparisons. The best-case time complexity is O(N).
  • Worst Case: The array is sorted in reverse order. Each element will need to be compared with all previous elements and moved to the beginning, resulting in a worst-case time complexity of O(N2).

Space Complexity of Insertion Sort

Insertion Sort works in-place and only requires constant space for the variables used during comparisons and swaps. Therefore, the space complexity of Insertion Sort is O(1).

Stability of Insertion Sort

Insertion Sort is a stable sorting algorithm. This means that if two elements have the same value, their relative order in the original array will be preserved after sorting. This is because elements are only shifted when necessary, and no unnecessary swaps are made.

Stability is crucial in cases where the order of equal elements needs to be maintained, especially when sorting based on multiple criteria. Insertion Sort preserves this property and ensures that the sorting process does not alter the relative order of equal elements.
