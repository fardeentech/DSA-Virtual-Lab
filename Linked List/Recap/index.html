Recap of Linked Lists

Overview

A Linked List is a linear data structure that consists of a sequence of elements, where each element (or node) contains a value and a reference (or pointer) to the next node in the sequence. Linked Lists are dynamic in size, allowing for efficient insertions and deletions compared to arrays.

Key Characteristics

  • Type: Linear data structure
  • Time Complexity:
    • Access: O(n) (to traverse to a particular element)
    • Search: O(n) (in the worst case, need to traverse the entire list)
    • Insertion: O(1) (at the head or tail), O(n) (at a given position)
    • Deletion: O(1) (if node is known), O(n) (if node needs to be searched)
  • Space Complexity: O(n) (for storing n elements)
  • Dynamic Size: Yes, can grow or shrink as needed.
  • Non-contiguous: Nodes may not be stored in contiguous memory locations.

How It Works

In a linked list, each node contains data and a reference to the next node. The first node is called the head, and the last node points to null (or None), indicating the end of the list. This structure allows for efficient memory usage and dynamic size adjustments.

Types of Linked Lists

  • Singly Linked List: Each node points to the next node, allowing for one-way traversal.
  • Doubly Linked List: Each node has references to both the next and previous nodes, allowing for bidirectional traversal.
  • Circular Linked List: The last node points back to the first node, forming a circular structure.

Example

Given the linked list: 1 → 2 → 3 → 4 → 5

After inserting a node with value 6 at the end, the list will be: 1 → 2 → 3 → 4 → 5 → 6

Use Cases

Linked Lists are particularly useful for:

  • Dynamic memory allocation where size is unpredictable
  • Implementing data structures like stacks and queues
  • When frequent insertions and deletions are needed
  • Maintaining ordered collections of items
